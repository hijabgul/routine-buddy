## 📖 Overview

Routine Buddy is a simple yet effective web-based application designed to help you organize and manage your daily, weekly, or monthly routines. Built with vanilla JavaScript, HTML, and CSS, it offers a straightforward interface to add, track, and complete your tasks without any complex setups. All your routines are saved directly in your browser's local storage, ensuring your data persists across sessions.

## ✨ Features

-   🎯 **Add New Routines**: Easily add new tasks or habits to your list.
-   ✅ **Mark as Complete**: Toggle the completion status of your routines.
-   🗑️ **Delete Routines**: Remove routines that are no longer needed.
-   ✏️ **Edit Routines**: Update the description of existing routines.
-   💾 **Local Data Persistence**: Routines are automatically saved to your browser's `localStorage` and persist between visits.
-   📱 **Responsive Design**: User interface adapts gracefully to various screen sizes, from desktop to mobile.

## 🚀 Quick Start

This project is a static web application that runs directly in your browser. There is no server-side component or complex build process required.

### Prerequisites

-   A modern web browser (e.g., Chrome, Firefox, Safari, Edge)

### Installation

1.  **Clone the repository**
    ```bash
    git clone https://github.com/hijabgul/routine-buddy.git
    cd routine-buddy
    ```

2.  **Open in your browser**
    Simply open the `index.html` file in your preferred web browser:
    ```bash
    # On most systems, you can simply run:
    open index.html
    # Or manually navigate to the file in your file explorer and double-click it.
    ```
    The application will load directly from the local file system.

## 🔧 Development

Development involves directly editing the `index.html`, `style.css`, and `app.js` files. Any changes made will reflect when you refresh the `index.html` page in your browser.

## 🚀 Deployment

To deploy this application, you can simply host the `index.html`, `style.css`, and `app.js` files on any static file hosting service (e.g., GitHub Pages, Netlify, Vercel, Firebase Hosting, Apache/Nginx).

### Example: Deploying with GitHub Pages

1.  Push your code to a GitHub repository (which you've already done!).
2.  Go to your repository settings on GitHub.
3.  Navigate to the "Pages" section.
4.  Under "Build and deployment", select `Deploy from a branch`.
5.  Choose your `master` branch (or whichever branch contains your `index.html`) and `/ (root)` for the folder.
6.  Save, and GitHub Pages will deploy your site. Your site will typically be live at `https://[your-username].github.io/[repository-name]/`.
